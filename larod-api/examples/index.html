larod-client: run a model from files
Validate the model, backend, input bytes, and SSD output files before writing C code.

This example does not use camera frames. It proves that the model loads and that the output tensor order matches your postprocessing assumptions.

# List available larod backends on the camera
larod-client -a

# Run one SSD MobileNet inference from raw RGB input bytes
larod-client \
  -d \
  -c axis-a8-dlpu-tflite \
  -g ssd_mobilenet_v1_coco_quant_postprocess.tflite \
  -R 1 \
  -i dog.bin \
  -o test_out0.bin \
  -o test_out1.bin \
  -o test_out2.bin \
  -o test_out3.bin
