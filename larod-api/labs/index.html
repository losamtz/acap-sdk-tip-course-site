Repository-Based Labs

These labs follow the Larod workshop sequence. Each one keeps the previous concepts and adds one new layer: manual inference, live VDO frames, preprocessing, non-blocking DMA-BUF reuse, and bbox visualization.

Lab 1: Validate Model Execution
Use larod-client to list backends, load a model, run inference from a prepared input tensor, and inspect output files.
Beginner
35 min

Learning Objectives

  • Confirm that the larod service is reachable on the camera
  • Identify the correct backend name for the target hardware
  • Run an SSD MobileNet model from raw RGB input bytes
  • Understand the four SSD output tensors before writing C postprocessing
Final Capability

After the final lab, the app should capture frames with VDO, pass frame-backed tensors into larod without a full-frame copy, run optional cpu-proc preprocessing, execute an SSD model, parse mapped output tensors, and draw detections with the bbox API.
