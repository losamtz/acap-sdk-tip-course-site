Workshop Structure
Angular source and packaged ACAP applications are separate workshop folders

Angular source projects

  • • `web-proxy-angular/acap-angular-ui`
  • • `web-proxy-angular/acap-angular-ui-routing`
  • • Run `npm install` and `npm run build` here
  • • Produces static browser files in `dist/.../browser`

ACAP package projects

  • • `web-proxy-angular/web-proxy-angular`
  • • `web-proxy-angular/web-proxy-angular-route`
  • • C backend starts CivetWeb on `localhost:2001`
  • • `app/html/` contains the Angular build output
Build the Angular Project
Compile the frontend before building the ACAP package
cd ~/Dev/projects/acaps/axis-acap-tip-workshop/webserver-proxy/web-proxy-angular/acap-angular-ui
npm install
npm run build

Important angular.json setting

"options": {
  "baseHref": "./",
  "browser": "src/main.ts",
  "polyfills": ["zone.js"]
}

`baseHref: "./"` keeps generated asset URLs relative to `index.html`, which is important when the page is served from the camera UI rather than from `/`.

Generated files

  • • `index.html`
  • • `main-*.js`
  • • `polyfills-*.js`
  • • `styles-*.css`
  • • static assets such as `Axis_Communications.svg`
