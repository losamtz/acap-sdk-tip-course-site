Workshop source

The main lab follows vapix/dynamic-overlay-vapix. The ONVIF sample is included as a secondary extension because it uses the same authenticated local HTTP pattern with a SOAP payload.

GitHub: axis-acap-tip-workshop/vapix
Lab 1: Dynamic Overlay with VAPIX
Build an ACAP application that creates a dynamic overlay through a JSON VAPIX request.
Primary - 80 min
Learning objectives
  • - Understand service-account authentication for ACAP applications.
  • - Implement local VAPIX requests with libcurl.
  • - Build and parse JSON using Jansson.
  • - Extract the returned overlay identity for later use.
Expected flow
  • - Manifest grants access to GetCredentials.
  • - D-Bus returns service-account credentials.
  • - Jansson creates an addText request.
  • - libcurl posts to dynamic overlay VAPIX.
Lab completion guide
Practical checks before considering the VAPIX exercises complete.

Success tips

  • Complete dynamic overlay before ONVIF.
  • Validate manifest permissions before debugging HTTP code.
  • Use curl verbose output only for non-secret diagnostics.

Common issues

  • Missing GetCredentials permission in manifest.json.
  • Proxy settings interfering with 127.0.0.12 requests.
  • JSON response parsed without first checking the VAPIX error field.
