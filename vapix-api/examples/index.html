Dynamic Overlay VAPIX
The primary workshop path: create a dynamic text overlay by posting JSON to a local VAPIX endpoint.
What this example teaches
  • - Request VAPIX service-account access in manifest.json.
  • - Retrieve credentials through D-Bus at runtime.
  • - POST to http://127.0.0.12 with libcurl.
  • - Build and parse JSON with Jansson.
  • - Read the returned overlay identity from the response.
Important details
  • - Do not log the service-account password.
  • - Use CURLOPT_NOPROXY, "*" for local requests.
  • - Check HTTP status and JSON error fields.
  • - Free JSON, curl, GLib, and credential resources.
D-Bus permission
The application must request access to VAPIX service-account credentials.
{
    "schemaVersion": "2.0.0",
    "resources": {
        "dbus": {
            "requiredMethods": [
                "com.axis.HTTPConf1.VAPIXServiceAccounts1.GetCredentials"
            ]
        }
    },
    "acapPackageConf": {
        "setup": {
            "friendlyName": "Dynamic Overlay Vapix",
            "appName": "dynamic_overlay_vapix",
            "runMode": "never",
            "version": "1.0.0"
        }
    }
}
Build Configuration
Libraries needed by the dynamic overlay example
PKGS = glib-2.0 gio-2.0 libcurl jansson

CFLAGS += $(shell pkg-config --cflags $(PKGS))
LDLIBS += $(shell pkg-config --libs $(PKGS))
GLib for utility types
GIO for D-Bus
libcurl for HTTP
Jansson for JSON
