What is the FastCGI API?

A high-performance web interface protocol that enables ACAP applications to handle HTTP requests through Apache web server integration. FastCGI provides persistent process execution, efficient request handling, and seamless integration with the camera's web interface, making it ideal for building responsive web APIs and configuration interfaces.

What is FastCGI?

FastCGI is a protocol for interfacing interactive programs with a web server. Unlike traditional CGI, FastCGI processes remain running between requests, providing significant performance improvements and resource efficiency.

Performance Benefits

  • • Persistent processes reduce startup overhead
  • • Connection pooling and request multiplexing
  • • Lower memory footprint than traditional CGI
  • • Faster response times for web requests

Integration Advantages

  • • Uses existing Apache web server
  • • Same authentication as VAPIX
  • • Unified user management
  • • Standard HTTP port (80/443)

Socket Communication

FastCGI uses Unix domain sockets for communication between Apache and your ACAP application:

# Socket path from environment variable
FCGI_SOCKET_NAME="/tmp/fcgi_socket"
# Socket permissions
chmod(socket_path, S_IRWXU | S_IRWXG | S_IRWXO);

HTTP Request Processing
Understanding how FastCGI processes HTTP requests, parses JSON payloads, and generates responses

Request Processing Flow

1
Accept Request

FCGX_Accept_r() waits for incoming HTTP requests from Apache

2
Extract Headers

Parse REQUEST_METHOD, SCRIPT_NAME, and other CGI environment variables

3
Route Request

Match script path and method to appropriate handler function

4
Process & Respond

Handle business logic, interact with ACAP APIs, and return JSON response

Main Processing Loop
// Main request handling loop
while (FCGX_Accept_r(&req) == 0) {
    syslog(LOG_INFO, "FCGX_Accept_r OK");
    
    // Route and handle the request
    route_request(&req, handle);
    
    // Finish request processing
    FCGX_Finish_r(&req);
}

FastCGI API Functions
Essential FastCGI library functions with practical code examples for request handling and response generation

Library Initialization Functions

FCGX_Init()

Initialize FastCGI library - must be called first

// Initialize FastCGI library
status = FCGX_Init();
if (status != 0) {
    syslog(LOG_ERR, "FCGX_Init failed");
    return status;
}
FCGX_OpenSocket()

Create listening socket from environment variable

// Get socket path from environment
char* socket_path = getenv("FCGI_SOCKET_NAME");
if (!socket_path) {
    syslog(LOG_ERR, "Failed to get FCGI_SOCKET_NAME");
    return EXIT_FAILURE;
}

// Open socket with backlog of 5
int sock = FCGX_OpenSocket(socket_path, 5);
chmod(socket_path, S_IRWXU | S_IRWXG | S_IRWXO);
FCGX_InitRequest()

Initialize request structure for handling

// Initialize request structure
FCGX_Request req;
int status = FCGX_InitRequest(&req, sock, 0);
if (status != 0) {
    panic("FCGX_InitRequest failed");
}
