ax_parameter_new
Create a new parameter handle for the specified application.

Function Signature

AXParameter* ax_parameter_new(const gchar* app_name, GError** error);

Usage Example

AXParameter* handle = ax_parameter_new("my_app", &error);
if (handle == NULL) {
    g_error("Failed to create parameter handle: %s", error->message);
}
ax_parameter_free
Free the parameter handle and all associated resources.

Function Signature

void ax_parameter_free(AXParameter* param);

Usage Example

ax_parameter_free(handle);
Usage Guidelines
Best practices and important considerations when using Parameter API functions

Initialization Pattern

Always create only one AXParameter instance per application and share it globally. Multiple instances can lead to unexpected behavior.

Callback Safety

Never call ax_parameter_* functions from within parameter callbacks to avoid deadlocks. Use global state and process updates in your main loop instead.

Memory Management

Always call ax_parameter_free() during application cleanup to properly release resources and prevent memory leaks.

Parameter Persistence

Parameters added via ax_parameter_add() are automatically persistent across application restarts, device reboots, and firmware upgrades.
