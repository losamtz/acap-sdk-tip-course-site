Event Producer Overview

An Event Producer is an ACAP application that creates and sends events to notify other applications about state changes, sensor readings, or custom data. Producers follow this workflow:

Declare Events: Register event types with the system
Create Events: Build event objects with data payload
Send Events: Transmit events to interested subscribers

ACAP Event Declaration Flow

Step-by-step process for declaring and sending events in ACAP SDK

1
Create Event Handler
Initialize the main event handler instance
app_data->event_handler = ax_event_handler_new();
2
Create Key-Value Set
Create container for event metadata and structure
key_value_set = ax_event_key_value_set_new();
3
Define Topic Hierarchy
Build the three-level topic hierarchy for event organization
// TOPIC LEVEL 0 (CameraApplicationPlatform)
ax_event_key_value_set_add_key_value(key_value_set,
    "topic0", "tnsaxis", TOPIC0_TAG, AX_VALUE_TYPE_STRING, NULL);

// TOPIC LEVEL 1 (SendPulse)  
ax_event_key_value_set_add_key_value(key_value_set,
    "topic1", "tnsaxis", TOPIC1_TAG, AX_VALUE_TYPE_STRING, NULL);
ax_event_key_value_set_add_nice_names(key_value_set,
    "topic1", "tnsaxis", TOPIC1_TAG, TOPIC1_NAME, NULL);

// TOPIC LEVEL 2 (SendPulseEvent)
ax_event_key_value_set_add_key_value(key_value_set,
    "topic2", "tnsaxis", EVENT_TAG, AX_VALUE_TYPE_STRING, NULL);
4
Add Event Data
Define the actual data payload that will be sent with events
ax_event_key_value_set_add_key_value(key_value_set,
    "value", NULL, &start_value, AX_VALUE_TYPE_INT, NULL);
ax_event_key_value_set_mark_as_data(key_value_set, 
    "value", NULL, NULL);
5
Declare Event
Register the event with the system (1 = stateless, 0 = stateful)
ax_event_handler_declare(event_handler, key_value_set, 1, 
    &declaration, declaration_complete, &start_value, NULL);
6
Setup Timer & Send Events
Start periodic event sending after declaration is complete
// In declaration_complete callback:
app_data->timer = g_timeout_add_seconds(5, 
    (GSourceFunc)send_event, app_data);
Key Concepts

Event States:

  • • Stateless (1): Fire-and-forget events
  • • Stateful (0): Events with persistent state

Topic Hierarchy:

  • • topic0: Platform level (CameraApplicationPlatform)
  • • topic1: Application level (SendPulse)
  • • topic2: Event level (SendPulseEvent)

gboolean ax_event_handler_declare(AXEventHandler* event_handler, AXEventKeyValueSet* key_value_set, gint stateless, guint* declaration, AXDeclarationCompleteCallback callback, gpointer user_data, GError** error)

Practical examples of implementing event callbacks in ACAP applications

Basic Application Event Callback
basic
Simple callback for handling application lifecycle events
// Callback function signature
static void on_application_event(guint subscription, AXEvent *event, gpointer user_data) {
    // Get the event type from the key-value set
    const gchar *event_key = ax_event_get_key_value_set(event);
    
    // Extract specific data from the event
    const gchar *app_name = ax_event_key_value_get(event, "application");
    const gchar *timestamp = ax_event_key_value_get(event, "timestamp");
    
    g_print("Application Event: %s\n", event_key);
    g_print("Application: %s\n", app_name ? app_name : "Unknown");
    g_print("Timestamp: %s\n", timestamp ? timestamp : "Not provided");
}

// Register the callback
AXEventHandler *handler = ax_event_handler_create(
    "tnsaxis:CameraApplicationPlatform/Application/Started",
    on_application_event,
    NULL,  // user_data
    &error
);
Advanced Callback with User Data
advanced
Callback that uses user data to maintain application state
// Application context structure
typedef struct {
    gchar *app_name;
    gint event_count;
    GMutex mutex;
} AppContext;

// Thread-safe callback with user data
static void on_app_status_event(guint subscription, AXEvent *event, gpointer user_data) {
    AppContext *ctx = (AppContext *)user_data;
    
    // Lock for thread safety
    g_mutex_lock(&ctx->mutex);
    
    const gchar *status = ax_event_key_value_get(event, "status");
    const gchar *message = ax_event_key_value_get(event, "message");
    
    ctx->event_count++;
    
    g_print("[%s] Event #%d - Status: %s\n", 
            ctx->app_name, ctx->event_count, status);
    
    if (message) {
        g_print("Message: %s\n", message);
    }
    
    g_mutex_unlock(&ctx->mutex);
}

// Setup with user data
AppContext *app_ctx = g_new0(AppContext, 1);
app_ctx->app_name = g_strdup("MyACAPApp");
app_ctx->event_count = 0;
g_mutex_init(&app_ctx->mutex);

AXEventHandler *handler = ax_event_handler_create(
    "tnsaxis:CameraApplicationPlatform/MyApp/StatusUpdate",
    on_app_status_event,
    app_ctx,  // Pass context as user_data
    &error
);

Robust callback with comprehensive error handling
// Error-aware callback function
static void on_license_event(guint subscription, AXEvent *event, gpointer user_data) {
    if (!event) {
        g_warning("Received NULL event in callback");
        return;
    }
    
    const gchar *event_key = ax_event_get_key_value_set(event);
    if (!event_key) {
        g_warning("Event has no key-value set");
        return;
    }
    
    // Check if this is a license event
    if (!g_str_has_prefix(event_key, "tnsaxis:CameraApplicationPlatform/License/")) {
        g_warning("Unexpected event type: %s", event_key);
        return;
    }
    
    const gchar *license_status = ax_event_key_value_get(event, "status");
    const gchar *app_id = ax_event_key_value_get(event, "application_id");
    const gchar *error_msg = ax_event_key_value_get(event, "error_message");
    
    if (g_str_has_suffix(event_key, "/Invalid") || g_str_has_suffix(event_key, "/Expired")) {
        g_critical("License issue for app %s: %s", 
                  app_id ? app_id : "unknown",
                  error_msg ? error_msg : "No details provided");
        
        // Handle license failure
        handle_license_failure(app_id, error_msg);
    } else if (g_str_has_suffix(event_key, "/Valid")) {
        g_info("License validated successfully for app: %s", 
               app_id ? app_id : "unknown");
    }
}

// Helper function for license failure handling
static void handle_license_failure(const gchar *app_id, const gchar *error_msg) {

    g_warning("Application %s license failure: %s", app_id, error_msg);
    
    // Could trigger application shutdown, send notification, etc.

}
Callback Best Practices

Do:

  • • Always validate event and user_data parameters
  • • Use thread-safe operations (mutexes, atomic operations)
  • • Keep callback functions lightweight and fast
  • • Free any allocated memory in callbacks
  • • Log important events for debugging

Don't:

  • • Block or perform long-running operations
  • • Call ax_event_handler_free() from within callback
  • • Assume event data will persist after callback returns
  • • Ignore error conditions or NULL pointers
  • • Access shared data without proper synchronization

Event Sending Function

Creates and sends pulse events with incrementing values, maintaining timer-based execution.

static gboolean send_event(AppData *send_data) {
    AXEventKeyValueSet *key_value_set = NULL;
    AXEvent  *event                   = NULL;
    
    key_value_set = ax_event_key_value_set_new();

    ax_event_key_value_set_add_key_value( key_value_set, "value", NULL, &send_data->value, AX_VALUE_TYPE_INT, NULL);
    
    // Create the event
    // Use ax_event_new2 since ax_event_new is deprecated from 3.2
    event = ax_event_new2(key_value_set, NULL);
    
    // The key/value set is no longer needed
    ax_event_key_value_set_free(key_value_set);

    if(!ax_event_handler_send_event(send_data->event_handler, send_data->event_id, event, NULL))
      LOG_ERROR("Could not fire event\n");
    ax_event_free(event);

    // Toggle value
    send_data->value = send_data->value >= 100 ? 0 : send_data->value + 10;
    
    // Returning TRUE keeps the timer going
    return TRUE;
}

  • Complexity: Single value vs multi-parameter events
Producer Best Practices

Event Design:

  • • Use meaningful topic names that describe the event
  • • Include relevant data in event payload
  • • Keep event frequency reasonable to avoid system overload
  • • Use appropriate data types (int, double, string)

Implementation:

  • • Always check return values from API functions
  • • Free event objects after sending to prevent memory leaks
  • • Handle declaration callbacks properly
  • • Use timers for periodic events, not busy loops
Common Pitfalls to Avoid

Declaration Issues:

  • • Don't send events before declaration is complete
  • • Don't use custom namespaces (always use "tnsaxis")
  • • Don't forget to add nice names for web UI visibility
  • • Don't skip marking data fields with mark_as_data()

Runtime Issues:

  • • Don't send events too frequently (causes system load)
  • • Don't forget to free event objects after sending
  • • Don't ignore error return values from API calls
  • • Don't use blocking operations in declaration callbacks
